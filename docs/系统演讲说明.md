# AI 销售培训系统 — 演讲用说明文档

面向「产品演示 / 技术分享 / 答辩」的提纲式说明，可与现场 Demo 配合使用。

---

## 一、系统定位

- **名称**：AI 销售通（仓库名：`ai-sales-training`）。
- **目标**：帮助销售人员在**知识巩固**与**邀约话术演练**两类场景下练习；部分能力依赖 **MiniMax** 等大模型与语音能力。
- **典型用户**：一线销售、培训讲师、运营验收 Demo。

---

## 二、功能模块概览

| 模块 | 路由 | 说明 |
|------|------|------|
| 首页 | `/` | 入口导航：基础知识训练、邀约情景模拟、外链「录音分析」等。 |
| 基础知识闯关练 | `/training`、`/training/[id]` | 从 Excel 题库抽题（销售 FAQ / 产品基础），**文字 + 语音作答**；语音经服务端 **STT** 转写；可逐题提交、提前结束；本地按要点匹配给出 **9/7/5/3 分档** 与培训总结。 |
| 邀约情景模拟 | `/practice/new`、`/practice/[id]` | 配置客户渠道、年级、成绩画像等，创建会话；与 AI 多轮对话（文本 / 录音+转写 / 实时房间等模式）；按对话**节点**（开场、挖需求、提信心等）计分与反馈；可生成报告页。 |
| 历史记录 | `/history` | 查看过往**演练**会话列表与成绩（依赖 `/api/sessions`）。 |
| 录音分析 | 外链 | 首页卡片跳转外部系统，非本仓库实现。 |

**培训**与**演练**两套会话体系并存：培训走 `trainingStore` + Excel；演练走 `sessions` 相关 API + 提示词与评分配置（`src/lib/prompts.ts`、`src/types` 等）。

---

## 三、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | **Next.js 16**（App Router） | 全栈：页面、Route Handlers、SSR/流式与客户端组件混用。 |
| 语言 | **TypeScript 5** | 类型约束 API 与领域模型。 |
| UI | **React 19**、**Tailwind CSS 4** | 响应式布局、移动端底栏等。 |
| 数据 | **xlsx** | 服务端读取 `src/data/tables` 下 Excel 题库；需 `serverExternalPackages` 避免被打包器错误处理。 |
| AI / 语音 | **MiniMax**（项目内封装于 `src/lib/minimax` 等） | 对话、TTS、**语音转文字**等（具体以环境变量与实现为准）。 |
| 持久化 | **JSON 文件**（如培训会话） | 轻量本地/单机部署友好；生产需评估并发与备份策略。 |

首页脚注可概括为：**Next.js + Tailwind + MiniMax**（与实现一致）。

---

## 四、架构与数据流（简述）

1. **基础知识训练**  
   `POST /api/training/sessions` → 读 Excel、洗牌、建会话 → 返回首题及标准答案等 → 前端带 Query 进入 `/training/[id]` → 用户作答 → `POST .../message` 推进题序与状态 → 总结页用**本地规则**对照标准答案要点打分。

2. **邀约演练**  
   `POST /api/sessions` 创建会话 → `/practice/[id]` 拉取会话与历史 → `POST .../message` 调用大模型续写对话并可能返回评分 → 结束与报告路由汇总结果。

3. **语音**  
   浏览器 **MediaRecorder** 采集（`useVoiceRecording`）→ `POST /api/speech-to-text` → 服务端调 STT → 回填输入框。

## 五、重难点（适合在演讲中展开）

### 5.1 工程与构建

- **xlsx 与 Next/Turbopack**：`xlsx` 依赖 Node 文件系统，需在 `next.config` 中配置 **`serverExternalPackages: ['xlsx']`**，否则打包/运行易出错。  
- **静态资源追踪**：部署时 Excel 路径通过 **`outputFileTracingIncludes`** 纳入 API 路由追踪，避免生产找不到题库文件。  
- **局域网调试**：`allowedDevOrigins` 便于手机连同一 WiFi 访问开发机端口（如 3040）。

### 5.2 前端体验与一致性

- **`useSearchParams` 与 Hydration**：查询串参与首屏时，服务端与客户端可能不一致；采用 **Suspense 边界**包裹使用查询参数的子树，避免 React Hydration 报错。  
- **首题标准答案传递**：培训首题若未把标准答案带到前端，本地评分会走「无标准答案 → 固定分」分支；需 **API + URL（或后续改为短链/GET）** 与设计对齐。  
- **语音**：各浏览器 MIME 与 iOS Safari 差异大，需在 **`MediaRecorder` 选型、分片 timeslice、Blob 类型** 上兼容（见 `useVoiceRecording`）。

### 5.3 业务与评分

- **双轨业务**：培训（Excel + 题序状态机）与演练（提示词 + 节点状态机）**模型不同**，讲解时避免混为一谈。  
- **评分语义**：培训总结中「总分 / 等级 / 档位色」需与 **9/7/5/3 分档**一致；若用「总分÷题数×10」划字母等级，会与档位语义冲突，已通过 **满分 9 分制 + 平均分定级** 等方式对齐（以当前代码为准）。  
- **Excel**：`sheet_to_json` 默认第一行为表头；若表头重复成行，需 **过滤「题干+答案等于列名」的假数据行**。

### 5.4 产品与运维

- **密钥与环境变量**：MiniMax 等密钥仅放服务端，避免暴露给浏览器。  
- **JSON 会话存储**：适合演示与小规模；若上生产需考虑 **文件锁、多实例、备份与迁移**。

## 六、Demo 建议顺序（约 10～15 分钟）

1. 首页 → 进入 **基础知识训练**，展示题型选择与一题作答（含语音转写若环境可用）。  
2. 打开 **培训总结**，说明分档与等级含义。  
3. **新建邀约演练** → 选画像 → 对话 2～3 轮 → 查看节点评分或报告。  
4. **历史记录** 一带而过。  
5. 技术向听众：展示 `next.config` 中 xlsx、文件追踪与 `trainingStore` 读表一行日志（可选）。

---

## 七、可主动提及的「局限 / 后续」

- 培训评分当前以**关键词覆盖**为主，非大模型裁判，可与「引入 LLM 判分」作为演进方向。  
- 标准答案过长时放 URL 可能触达长度上限，可演进为 **sessionStorage 或专用 GET**。  
- 演练与培训的数据存储方式不统一，若产品化可收敛为统一后端与数据库。
