## 1. 产品概述

### 1.1 产品定位

AI 销售话术陪练系统：让 AI 扮演家长/学生角色，与销售人员进行**实时语音对话**演练，演练结束后系统根据预设评分标准给出综合评分和改进建议。

### 1.2 核心价值

- **新人入职陪练**：快速熟悉销售话术，缩短上手周期
- **场景化演练**：按客户类型（成绩好/中/差）定制对话策略
- **客观量化打分**：消除人工评课主观性，支持横向对比
- **数据驱动改进**：定位薄弱环节，持续优化话术

## 3. 演练流程设计

### 3.1 标准演练流程

```
[1. 选择场景] → [2. AI开场] → [3. 多轮对话] → [4. 演练结束] → [5. 生成报告]
```

| 阶段 | 时长 | AI扮演 | 考核重点 |
|------|------|--------|---------|
| 开场 | 1-2轮 | 家长/学生 | 说明来意（15分） |
| 探需 | 3-5轮 | 家长/学生 | 挖需求（25分） |
| 增强信心 | 2-3轮 | 家长/学生 | 提信心（15分） |
| 举案例 | 1-2轮 | 家长/学生 | 举例（20分） |
| 给方案 | 1-2轮 | 家长/学生 | 给方案（15分） |
| 邀约确认 | 1-2轮 | 家长/学生 | 邀约确认（10分） |

### 3.2 客户类型池

| 类型 | 标签 | 典型特征 | AI开场策略 |
|------|------|---------|-----------|
| 成绩优秀型 | `type_a` | 目标高、选老师挑 | "你们师资比别家强在哪？" |
| 成绩中等型 | `type_b` | 犹豫不决、要案例 | "有没有和我家孩子情况类似的？" |
| 成绩较差型 | `type_c` | 焦虑、担心花钱无效 | "真能提分吗？多久见效？" |
| 时间紧张型 | `type_d` | 孩子忙、作业多 | "孩子没时间啊，放学都十点了" |
| 已报班型 | `type_e` | 在对比、想换机构 | "我们已经报了XX了" |

### 3.3 对话状态机

```
                     ┌──────────────┐
                     │   IDLE       │
                     │   待开始      │
                     └──────┬───────┘
                            │ 销售人员点击"开始"
                            ▼
                     ┌──────────────┐
          ┌─────────│  WAIT_AI     │  AI生成开场白
          │         │   等待AI发言  │──────────────┐
          │         └──────────────┘              │
          │                                        ▼
          │  销售人员                   ┌──────────────┐
          │  回复          ┌───────────│  AI_SPEAKING │◄─┐
          │ ◄──────────────│  AI发言中  │──────────────┘  │
          │                └────────────┘                  │
          │                      │                          │
          │                      │ AI说完/超时              │
          ▼                      ▼                          │
   ┌──────────────┐      ┌──────────────┐                  │
   │ WAIT_SALES   │─────▶│ WAIT_SALES   │──────────────────┘
   │  等待销售回复 │ 销售  │  等待销售回复 │  循环对话
   └──────────────┘      └──────────────┘
          │
          │ 触发结束条件
          ▼
   ┌──────────────┐
   │   FINISHED   │
   │   演练结束   │
   └──────────────┘
```

**结束条件**（满足任一即可）：
- 销售人员成功邀约到店（关键词触发）
- 对话超过 15 轮
- 销售人员主动结束
- 出现超时/异常

## 4. 评分系统

### 4.1 评分维度

| 维度 | 分值 | 考核点 |
|------|------|--------|
| 说明来意 | 15 | 自我介绍清晰、抓住当下需求 |
| 挖需求 | 25 | 询问分数/排名/弱科、按成绩分层应对 |
| 提信心 | 15 | 分析原因、针对性提升信心 |
| 举例 | 20 | 有真实案例（相似情况、变化过程） |
| 给方案 | 15 | 给出可行方案、邀约到店 |
| 邀约确认 | 10 | 确认具体时间、发送位置信息 |

### 4.2 分步打分机制

每个节点演练后**即时评分**，而非等结束后一次性打分：

```javascript
// 示例：节点完成后的评分反馈
{
  node: "挖需求",
  score: 18,
  maxScore: 25,
  feedback: "较好地询问了分数和排名，但未明确追问弱科原因",
  suggestions: [
    "建议追问：孩子这科听不懂是从什么时候开始的？",
    "建议确认：目前校内排名大概在什么位置？"
  ]
}
```

### 4.3 综合评分报告

```json
{
  "totalScore": 82,
  "grade": "B+",
  "dimensions": [
    { "name": "说明来意", "score": 13, "max": 15, "detail": "..." },
    { "name": "挖需求", "score": 18, "max": 25, "detail": "..." },
    { "name": "提信心", "score": 12, "max": 15, "detail": "..." },
    { "name": "举例", "score": 16, "max": 20, "detail": "..." },
    { "name": "给方案", "score": 13, "max": 15, "detail": "..." },
    { "name": "邀约确认", "score": 10, "max": 10, "detail": "..." }
  ],
  "highlight": "举案例环节表现优秀，能有效引用相似案例增强说服力",
  "weakness": "挖需求环节对弱科原因追问不够深入",
  "improvements": [
    "建议加强：孩子这科听不懂是从什么时候开始的？",
    "建议确认：目前校内排名大概在什么位置？"
```

## 6. API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/sessions` | 创建演练会话 |

| `POST` | `/api/sessions/:id/message` | 发送销售消息 |
| `GET` | `/api/sessions/:id/messages` | 获取对话历史 |
| `POST` | `/api/sessions/:id/end` | 结束演练 |
| `GET` | `/api/sessions/:id/report` | 获取评分报告 |
| `GET` | `/api/knowledge/faqs` | 获取FAQ列表 |
| `GET` | `/api/knowledge/scripts` | 获取话术模板 |
| `POST` | `/api/knowledge/faqs` | 新增FAQ（管理员） |

### 6.2 请求/响应示例

#### 创建演练会话

```typescript
// POST /api/sessions
// Request

  "salespersonId": "user_123",

  "aiOpeningAudio": "https://..."           // AI开场白（音频URL）
```

```typescript
// Response
{
  "aiMessage": "哦，那孩子这次考试数学大概多少分？",
  "aiAudio": "https://...",
  "scores": {
    "node": "说明来意",
```

## 7. 数据模型

### 7.1 数据库表结构

```sql
-- 销售人员表
CREATE TABLE salespersons (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  phone TEXT,
  role TEXT DEFAULT 'sales',  -- sales | admin | super_admin
  created_at TIMESTAMP DEFAULT NOW()
);

-- 演练会话表
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  salesperson_id TEXT REFERENCES salespersons(id),
  customer_type TEXT NOT NULL,      -- type_a/b/c/d/e
  customer_score TEXT,               -- 好/中/差
  customer_subject TEXT,            -- 弱科
  status TEXT DEFAULT 'active',     -- active | finished
  started_at TIMESTAMP DEFAULT NOW(),
  finished_at TIMESTAMP
);

-- 对话记录表
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  session_id TEXT REFERENCES sessions(id),
  role TEXT NOT NULL,               -- ai | sales
  content TEXT NOT NULL,            -- 文字内容
  audio_url TEXT,                   -- 音频URL
  node TEXT,                        -- 所处节点
  created_at TIMESTAMP DEFAULT NOW()
);

-- 评分记录表
CREATE TABLE scores (
  id TEXT PRIMARY KEY,
  session_id TEXT REFERENCES sessions(id),
  node TEXT NOT NULL,               -- 评分节点
  score INTEGER NOT NULL,
  max_score INTEGER NOT NULL,
  feedback TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 最终报告表
CREATE TABLE reports (
  id TEXT PRIMARY KEY,
  session_id TEXT REFERENCES sessions(id),
  total_score INTEGER,
  grade TEXT,
  dimension_scores JSONB,           -- 各维度分数详情
  highlight TEXT,                    -- 亮点
  weakness TEXT,                     -- 薄弱点
  improvements TEXT[],               -- 改进建议
  created_at TIMESTAMP DEFAULT NOW()
);

-- FAQ知识库表
CREATE TABLE faqs (
  id TEXT PRIMARY KEY,
  question TEXT NOT NULL,
  answer TEXT NOT NULL,
  category TEXT,                     -- 问题分类
  tags TEXT[],                       -- 标签
  created_at TIMESTAMP DEFAULT NOW()
);
```

## 9. AI Prompt 设计

### 9.1 系统角色设定

```markdown
# 角色：你是一个家长/学生

## 基本信息
- 你是客户的[孩子家长/学生本人]
- 孩子[成绩类型：优秀/中等/较差]
- 孩子[弱科科目]
- 你的身份背景：[职业/性格特点]

## 性格设定
- 如果家长是"成绩优秀型"：目标高、挑剔、喜欢比较
- 如果家长是"成绩中等型"：犹豫不决、需要案例说服、关心效果
- 如果家长是"成绩较差型"：焦虑、担心被骗、关心价格

## 话术原则
1. 不要主动暴露太多信息，让销售挖掘
2. 可以提出异议和质疑，测试销售应对能力
3. 遇到压力时可以有情绪反应（不耐烦、质疑）
4. 适当时机可以被说服，达成到店邀约

## 节点目标
- 开场：简单回应，观察销售自我介绍
- 挖需求：通过对话自然透露孩子情况
- 提信心：测试销售是否能打消顾虑
- 举案例：看销售是否能给出有力案例
- 给方案：测试销售是否能给出可行方案
- 邀约确认：最终是否能约定到店时间
```

### 9.2 评分Prompt模板

```markdown
# 评分任务

## 当前节点：{node_name}
## 满分：{max_score}

## 销售的回答：
"{sales_response}"

{evaluation_criteria}

## 打分标准：
{exact_standards}

1. 是否覆盖了所有考核点
2. 回答的专业性和说服力

输出JSON格式：
```

## 12. 风险与挑战

| 风险 | 应对方案 |
|------|---------|
| MiniMax API稳定性 | 预留备选方案（如有需） |
| 对话流程难以覆盖所有场景 | 先做P0场景覆盖，碎片场景用兜底话术 |
| 评分主观性 | 评分标准尽量细化，减少歧义 |
| 销售人员不配合使用 | 管理层推动 + 激励奖励机制 |
